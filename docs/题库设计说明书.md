# NPFJ 题库设计说明书

> 用于向 AI（Gemini、ChatGPT 等）描述整个题库架构，让其根据每个题池的特征逐池生成 5 道题。

---

## 一、系统概述

NPFJ 是一个**动态二分树路由**人格测评系统，共 16 个题池、80 道题（每池 5 题）。与传统 MBTI "统一答题→统一算分→出结果"不同，NPFJ 的特点是：**每道题的选项决定了下一题去哪**。用户每走完一个阶段的 5 道题，系统根据多数决路由到下一阶段的某个题池，路径越来越细分，最终收敛到 16 种人格之一。

同时，每道题的选项背后隐藏了 **5 个心理学维度的权重分**，用户在答题时完全感知不到，最终累积生成一张**五维心理雷达图**。这 5 个维度是：

| # | 维度名 | 通俗含义 | 低分端 ← | → 高分端 |
|---|--------|---------|----------|---------|
| 1 | 精神续航 | 大脑需要什么奖励 | 高频刷新（多巴胺依赖） | 深度长效（内啡肽耐受） |
| 2 | 社交磁场 | 需要被看见的程度 | 同温层渴望（群体认同） | 单机沙盒（独立自洽） |
| 3 | 秩序洁癖 | 对环境的掌控欲 | 绝对掌控（秩序强迫） | 随机探索（拥抱混乱） |
| 4 | 压力解压阀 | 上网为了什么 | 赛博充电（建设性升华） | 情绪断联（宣泄逃避） |
| 5 | 脑力劳作 | 享受烧脑的程度 | 掘根刨底（深度思辨） | 直通结论（省力原则） |

## 二、四个阶段与人格维度

```
Stage 0: 基础校准（题池 0）—— 不分流向，纯热身，固定路由到 Stage 1

Stage 1: 连接拓扑（题池 1）
  左 L → N（Network 公网互联型）：喜欢开源、云端协作、在社区中获取能量
  右 R → S（Silo 深潜内网型）：偏爱本地部署、绝对掌控、独享环境深耕

Stage 2: 探针模式（题池 2/3）—— 根据 Stage 1 的结果分流
  左 L → P（Ping 主动发包型）：先做再说、破坏性测试、Vibe Coding
  右 R → L（Listen 端口监听型）：先观察再行动、搭建信息流、静默吸收

Stage 3: 路由逻辑（题池 4/5/6/7）—— 根据前两阶段的结果分流
  左 L → F（Forward 极速转发型）：直觉驱动、追求效率、快速出活
  右 R → R（Route 精准路由型）：严谨架构、底层逻辑、不搞清楚不罢休

Stage 4: 输出格式（题池 8/9/10/11/12/13/14/15）—— 最终决出
  左 L → J（JSON 结构化型）：秩序规整、模板化管理、清单控
  右 R → W（Wildcard 通配符型）：随性探索、沙盒模式、自由发挥
```

## 四、题池设计明细

### 题池 0：基准校准（5 题）

- **阶段**: 0
- **目标**: 让用户进入答题状态，建立心理安全感
- **特点**: 题目通用、轻松，不涉及任何专业术语。A/B 选项不区分好坏
- **路由**: 无论选什么，固定进入题池 1（不走路由判定）
- **建议话题**: 日常数字习惯、接触电子设备的频率、对社交媒体的基本态度
- **注意**: 这 5 题不参与路由判定，但权重分正常累加

---

### 题池 1：连接拓扑（5 题）——决定 N vs S

- **阶段**: 1
- **目标**: 测出用户偏好在"公网互联/开源协作"还是"本地私有/独立深耕"
- **话题范围**:
  - 遇到问题先搜索还是自己钻研
  - 喜欢使用云服务还是本地软件
  - 对开源社区的态度
  - 团队协作时的工作方式
  - 对新工具/新平台的尝试意愿
- **N 倾向题**: 场景涉及共享、协作、云端、社区、开放
- **S 倾向题**: 场景涉及独立、私密、本地、掌控、专属
- **权重倾向**: N 偏向社交磁场+1（需要与他人连接）；S 偏向秩序洁癖+1（需要掌控环境）

---

### 题池 2：探针模式（5 题）——走 N 路线的人测 P vs L

- **阶段**: 2
- **前置**: 用户已被判定为 N（公网互联型）
- **目标人群画像**: 习惯在公网获取能量的人，对新事物持开放态度，但测试他遇到新事物是"先上手干"还是"先观察吸收"
- **P 倾向题**: 场景涉及直接上手、先跑再说、破坏性测试、冲动尝试
- **L 倾向题**: 场景涉及先收集信息、搭建信息流、看完文档再操作
- **权重倾向**: P 偏向精神续航+1（即时满足驱动）；L 偏向脑力劳作+1（深度吸收驱动）

---

### 题池 3：探针模式（5 题）——走 S 路线的人测 P vs L

- **阶段**: 2
- **前置**: 用户已被判定为 S（深潜内网型）
- **目标人群画像**: 习惯在自己的世界里独立工作，私密性强。同样测试"先上手干"还是"先观察吸收"，但场景要贴合"独立工作者"的语境
- **话题差异**: 不涉及社区互动/开源话题，转为个人项目、本地环境、工具探索
- **P 倾向题**: 拿到新工具直接解压运行、遇到问题试错
- **L 倾向题**: 先看 README、搭建好环境再跑、系统性地研究

---

### 题池 4：路由逻辑（5 题）——走 N+P 路线的人测 F vs R

- **阶段**: 3
- **前置**: 公网互联 + 主动参与型
- **目标人群画像**: 在社区活跃、拿到东西先上手再说的人。测试他是"追求快速出活"还是"讲究底层逻辑"
- **话题范围**: 写代码/做项目时的风格、遇到 Bug 的处理方式、对技术深度的追求
- **F 倾向题**: 先跑通再说、遇到问题找绕过方案、追求产出
- **R 倾向题**: 必须弄清原理、遇到问题死磕源码、追求优雅
- **权重倾向**: F 偏向压力解压阀+1（完成为主）；R 偏向脑力劳作+2（深度思辨）

---

### 题池 5：路由逻辑（5 题）——走 N+L 路线的人测 F vs R

- **阶段**: 3
- **前置**: 公网互联 + 安静监听型
- **目标人群画像**: 在公网默默收集信息、不着急表态的人。场景要从"信息处理"切入——他是快速抓取关键信息还是深度消化所有信息
- **话题差异**: 阅读长文的习惯、对教程/文档的态度、处理信息时的优先级
- **F 倾向题**: 扫读、跳读、先拿结论再决定要不要细看
- **R 倾向题**: 从头到尾看完、做笔记、弄懂所有细节后才算"看完"
- **权重倾向**: 同上

---

### 题池 6：路由逻辑（5 题）——走 S+P 路线的人测 F vs R

- **阶段**: 3
- **前置**: 深潜内网 + 主动参与型
- **目标人群画像**: 在自己领域内积极产出、习惯独立工作的人。场景要从"个人项目/创作"切入——追求效率产出还是追求底层完美
- **话题范围**: 做项目时的决策方式、对工具的选择标准、时间管理
- **F 倾向题**: 用最熟悉的工具快速出成果、达到目的就满足
- **R 倾向题**: 花时间调优工具链、追求优雅的方案、死磕每一个细节

---

### 题池 7：路由逻辑（5 题）——走 S+L 路线的人测 F vs R

- **阶段**: 3
- **前置**: 深潜内网 + 安静监听型
- **目标人群画像**: 独立且安静的技术深耕者/研究人员。场景要从"学习/研究"切入——快速掌握新知识还是彻底吃透一个领域
- **话题范围**: 学习新技术的习惯、阅读技术文档的方式、对广度和深度的取舍
- **F 倾向题**: 需要什么学什么、快速掌握够用的知识面
- **R 倾向题**: 系统性地学完一个领域、基础必须扎实

---

### 题池 8：输出格式（5 题）——走 N+P+F 路线的人决出 J vs W

- **阶段**: 4
- **前置**: 公网互联 + 主动参与 + 效率直觉
- **目标人群画像**: 开源社区的活跃贡献者，追求快速产出，习惯在公网分享和协作。测的是他最后的管理方式——有条不紊还是随性而为
- **话题范围**: 项目管理习惯、对 TODO List 的态度、周末的安排方式、桌面/文件管理风格
- **J 倾向题**: Todoist/看板管理、文件归类整齐、计划先于行动
- **W 倾向题**: 想到了就做、桌面随性、计划赶不上变化、自由发挥
- **权重倾向**: J 偏向秩序洁癖+2；W 偏向社交磁场或精神续航（依据具体选项）

---

### 题池 9：输出格式（5 题）——走 N+P+R 路线的人决出 J vs W

- **阶段**: 4
- **前置**: 公网互联 + 主动参与 + 逻辑深度
- **目标人群画像**: 社区中的深度贡献者，喜欢讲逻辑、追原理。同样测最终输出风格——系统化输出还是随性发散
- **话题范围**: 写文档/写代码的风格、对知识管理的态度、长期规划习惯
- **J 倾向题**: 结构化输出、知识体系化、长期计划
- **W 倾向题**: 想到哪写到哪、笔记本杂乱但自己找得到、短期自由发挥

### 题池 10：输出格式（5 题）——走 N+L+F 路线的人决出 J vs W

- **目标人群画像**: 在公网安静地收集信息、快速处理、不主动输出的人。测他收集到的信息最后怎么处理——归档整理还是随取随用
- **话题范围**: 收藏夹管理、阅读后的习惯、遇到好文章怎么处理
- **J 倾向题**: 按标签/文件夹分类、定期整理、建立索引
- **W 倾向题**: 一键收藏不分类、随用随搜、信任搜索功能

## 五、每题 JSON 格式

```json
{
  "id": 1,
  "text": "题目文字（决不让用户感到被评判，都是中性的两难选择）",
  "options": {
    "A": {
      "text": "选项A的文字（偏左的选项，route 为该池的左方向）",
      "route": "L",
      "weights": [1, 0, 2, 0, 0]
    },
    "B": {
      "text": "选项B的文字（偏右的选项，route 为该池的右方向）",
      "route": "R",
      "weights": [0, 1, 0, 2, 1]
    }
  }
}
```

### weights 填写参考

| 权重位置 | 维度 | 什么时候 +1 或 +2 |
|----------|------|-----------------|
| [0] | 精神续航 | 选项涉及即时满足/刺激 → +1；涉及持续性满足 → 0 或 -（用其他维度代替） |
| [1] | 社交磁场 | 选项涉及他人认同/互动/群体归属 → +1 或 +2 |
| [2] | 秩序洁癖 | 选项涉及计划/整理/掌控 → +1 或 +2 |
| [3] | 压力解压阀 | 选项涉及情绪宣泄/逃避/释放压力 → +1 或 +2 |
| [4] | 脑力劳作 | 选项涉及深度思考/分析/钻研 → +1 或 +2 |

每个选项的 weights 总和建议在 2-4 之间。不需要每个维度都填分，只填与该选项心理学倾向最相关的 1-3 个维度。

## 六、出题原则

1. **A/B 选项必须真实两难**——不能一个明显好一个明显差，要让用户犹豫
2. **每池 5 题各有侧重**——不重复同样的情境，覆盖该维度的不同方面
3. **语言中性，不带贬义**——永远不要让用户觉得"选 A 是好的、选 B 是差的"
4. **贴合目标人群的心理场景**——题池 2（N 型人）的题目不能用在题池 3（S 型人），他们的生活语境完全不同
5. **Stage 0 的题目要最轻松**——激活答题状态，不涉及任何专业判断
6. **Stage 4 的题目要最精准**——直接命中"生活管理方式"这个核心
