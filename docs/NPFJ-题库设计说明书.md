# NPFJ 测评系统设计说明书

> 给组员看的完整机制说明

---

## 一、核心架构概览

NPFJ 是一个**动态二分树路由**人格测评系统。

与传统测试（全部答完→统一算分→出结果）不同，NPFJ 的机制是：

> **每道题的选项决定了下一题去哪个题池。用户每一步选择都在走独一无二的路径。**

### 二分树路由（1+2+4+8）

```
Stage 1 (1个题池) ───────────── 决定 I / R（能量表达）
         │
    ┌────┴────┐
    ▼         ▼
Stage 2 (2个题池) ──────────── 决定 P / S（参与方式）
    │         │
  ┌─┴─┐     ┌─┴─┐
  ▼   ▼     ▼   ▼
Stage 3 (4个题池) ──────────── 决定 A / C（认知方式）
  │   │   │   │
 ┌┴┐ ┌┴┐ ┌┴┐ ┌┴┐
 ▼ ▼ ▼ ▼ ▼ ▼ ▼ ▼
Stage 4 (8个题池) ──────────── 决定 D / F（表达方式）
  │   │   │   │   │   │   │   │
  ▼   ▼   ▼   ▼   ▼   ▼   ▼   ▼
         16 种人格之一
```

**题池数量：** 1 + 2 + 4 + 8 = **15 个题池**
**每题数量：** 每池 5 题
**总题数：** 15 × 5 = **75 题**

### 没有"基准测试"/"热身"

用户直接从 Stage 1 开始答题，没有额外的基准校准阶段。5 题决定第一维度方向后，直接路由到对应的 Stage 2 题池。

## 二、四个维度的含义

每个维度对应二分树的一个 Stage，5 题多数决决定方向。

| Stage | 维度 | 方向 L → | 方向 R → | 测的是什么 |
|-------|------|---------|---------|-----------|
| 1 | **I / R** 能量表达 | I = 外放冲动型 | R = 内收克制型 | 你在网络世界释放能量的方式 |
| 2 | **P / S** 参与方式 | P = 主动参与型 | S = 被动观察型 | 你与网络内容的互动深度 |
| 3 | **A / C** 认知方式 | A = 分析驱动型 | C = 语境驱动型 | 你的信息处理模式 |
| 4 | **D / F** 表达方式 | D = 深度表达型 | F = 碎片表达型 | 你的输出形态 |

**四位人格码 = Stage1字母 + Stage2字母 + Stage3字母 + Stage4字母**
例如：I + P + A + D = **IPAD**（逻辑爆破者）

## 四、横向解耦 + 纵向耦合

这是 NPFJ 最核心的设计哲学。

### 横向解耦（每个 Stage 只测一个维度）

```
Stage 1 的 1 个题池（Pool 1）→ 只测 I/R
Stage 2 的 2 个题池（Pool 2/3）→ 只测 P/S
Stage 3 的 4 个题池（Pool 4-7）→ 只测 A/C
Stage 4 的 8 个题池（Pool 8-15）→ 只测 D/F
```

同一个 Stage 内的不同题池**横向上完全解耦**——题目场景可以不同（因为用户的前置路径不同），但测量的标尺一模一样，绝不混入其他维度的判断。

例如 Pool 2（I型人的 P/S 测试）和 Pool 3（R型人的 P/S 测试），题目内容完全不同（一个是外放者的参与方式，一个是内收者的参与方式），但判定标准都是"选 L 多 = P（参与）、选 R 多 = S（观察）"。

### 纵向耦合（路径决定结果）

用户的答题路径决定了最终走向哪个池、获得什么人格——

```
选 L → 走向左分支的题池
选 R → 走向右分支的题池

5 题走完后统计：L 多还是 R 多
多数决 → 路由到下一阶段对应题池
```

所以人格结果**不是算总分算出来的，而是路径走出来的**。这就是"纵向耦合"——系统的状态沿着二叉树的纵向路线一步步收敛。

## 五、隐性权重五维图（与人格判定完全解耦）

### 独立于人格的权重系统

每道题的 4 个选项（A/B/C/D）背后隐藏了 5 个心理学维度的权重值。

| 权重位置 | 维度名 | 含义 | 什么时候加分 |
|---------|--------|------|------------|
| [0] | 精神续航 | 大脑需要什么奖励 | 选项涉及即时满足/多巴胺驱动 |
| [1] | 社交磁场 | 需要被看见的程度 | 选项涉及他人认同/群体互动 |
| [2] | 秩序洁癖 | 对环境的掌控欲 | 选项涉及计划/整理/秩序 |
| [3] | 压力解压阀 | 上网为了什么 | 选项涉及情绪宣泄/逃避 |
| [4] | 脑力劳作 | 享受烧脑的程度 | 选项涉及深度思考/分析 |

**关键：** 这 5 个维度的累加**完全独立于路由判定**。路由走哪边由 route 标签决定，权重加多少由 weights 数组决定，两者互不干扰。

### 先验相关但不耦合

人格判定（4 位字母）和五维图（5 个雷达图轴）在**架构上是完全解耦的**——代码里没有任何"如果人格是 X，就把雷达图调到 Y"的逻辑。

但在心理现实中它们是**先验相关的**——一个人的底层心理动机（五维）决定了他的行为模式（人格），所以统计上会有聚类。那些打破了先验相关的"离群值"才是这个系统最有趣的部分。

## 六、每题格式说明

```json
{
  "id": 1,
  "text": "题目文字（四选一，让用户真的犹豫）",
  "options": {
    "A": { "text": "选项A（强烈偏左）", "route": "L", "weights": [2,0,1,0,0] },
    "B": { "text": "选项B（轻微偏左）", "route": "L", "weights": [1,1,0,0,0] },
    "C": { "text": "选项C（轻微偏右）", "route": "R", "weights": [0,0,1,1,0] },
    "D": { "text": "选项D（强烈偏右）", "route": "R", "weights": [0,0,0,2,1] }
  }
}
```

| 字段 | 说明 |
|------|------|
| `id` | 全局唯一题号（1-75） |
| `text` | 题目文字 |
| `options.A/B/C/D` | 四个选项 |
| `route` | "L" 或 "R"，决定二分树走向 |
| `weights[0..4]` | 五维权重，每个维度 0-2 分 |

**选项分布原则：**
- A / B → 都指向 L（左分支），A 程度更强
- C / D → 都指向 R（右分支），D 程度更强
- 四个选项让用户无法直接判断"选左还是选右"，必须区分程度的细微差别

## 八、75 道题的分布

| 阶段 | 维度 | 题池数 | 题数 | 说明 |
|------|------|--------|------|------|
| Stage 1 | I vs R | 1 | 5 | 所有人从这 5 题开始 |
| Stage 2 | P vs S | 2 | 10 | 按 I/R 分流，各 5 题 |
| Stage 3 | A vs C | 4 | 20 | 按 I/R+P/S 分流，各 5 题 |
| Stage 4 | D vs F | 8 | 40 | 按前三阶段路径分流，各 5 题 |
| **合计** | | **15** | **75** | |

一个用户实际只走 4 个题池 × 5 题 = **20 题**，但因为有树状分流，不同用户走到的题池和题目可以完全不同。
